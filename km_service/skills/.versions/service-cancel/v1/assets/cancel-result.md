# 退订结果回复模板

## 单个业务退订成功

适用条件：

- `cancel_service` 返回成功
- 用户有多个待退订业务（如停机保号前批量取消增值业务）

推荐话术骨架：

```text
{业务名称}已成功取消，将在{生效时间}正式生效。{退款说明}
```

示例：

```text
短信百条包已成功取消，将在次月1日00:00正式生效，当月费用不退，次月起将不再扣费。
```

## 还有待退订业务时（追加）

适用条件：

- 上一个退订已完成
- 用户仍有其他待退订业务

推荐话术骨架：

```text
{上一个业务退订结果}

您还订阅了以下增值业务：
- {业务名称1}（{月费}/月）
- {业务名称2}（{月费}/月）

请问需要继续取消哪个，还是先保留？
```

约束：

- 必须等 `cancel_service` 工具返回后再回复，禁止在回复中预判下一个退订的结果
- 禁止在同一轮回复中连续调用 `cancel_service` 两次
- 业务列表必须来自 `query_subscriber` 返回的数据，禁止自行编造

## 全部退订完成

适用条件：

- 所有用户选定的增值业务均已退订
- 或用户明确表示不再需要退订其他业务

推荐话术骨架：

```text
所有选定的增值业务均已取消，均将在次月1日00:00正式生效。如有其他问题，请随时告诉我。
```
