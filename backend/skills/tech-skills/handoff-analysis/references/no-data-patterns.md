# 工具空数据判定规则

## 用途

当工具调用成功但返回结果为"查无数据"时，需要在转人工摘要中准确标注为"无数据"而非"成功"。

## 匹配模式

以下中英文关键词出现在工具返回文本中时，判定为"空数据"：

- 没找到
- 未找到
- 不存在
- 没有...记录
- 无记录
- null
- not found

## 区分逻辑

| 工具返回状态 | 标注方式 |
|---|---|
| 调用失败（异常/超时） | 失败 |
| 调用成功 + 命中空数据模式 | 成功（无数据） |
| 调用成功 + 未命中空数据模式 | 成功 |

## 上下文参数

| 参数 | 值 | 说明 |
|---|---|---|
| 对话历史截取 | 最近 12 轮 | 平衡上下文丰富度与 LLM token 消耗 |
| 工具调用截取 | 最近 8 次 | 覆盖典型会话的工具调用链 |
| 工具结果摘要长度 | 前 150 字 | 在摘要卡片中可读的合理长度 |
