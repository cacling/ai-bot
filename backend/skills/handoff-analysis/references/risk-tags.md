# 风险标签定义

以下标签标识转人工会话中的特殊情况，供坐席提前准备应对策略。

| 标签 | 中文含义 | 判断依据 |
|------|---------|---------|
| `complaint` | 用户投诉 | 出现"投诉"、"举报"、"向上反馈"等词，或明确表示不满意服务质量 |
| `high_value` | 高价值用户 | 套餐费用超过 200 元/月，或账户余额/历史消费显著高于均值 |
| `churn_risk` | 流失风险 | 用户提到"取消"、"注销"、"换运营商"、"不用了"、"考虑换一家" |
| `overdue` | 账单逾期 | 账单状态为 overdue，或用户提到欠费、被停机 |
| `repeated_contact` | 多次联系 | 用户提到"之前联系过"、"已经说了好几次"，或同一问题在对话中反复出现 |
| `angry` | 情绪激烈 | 使用强烈否定词、感叹词，语气明显急躁或愤怒，或有言语攻击性 |
| `high_risk_op` | 高风险操作 | 涉及销户、实名变更、大额退款、套餐大幅降级等不可逆操作 |

## 使用规则
- 可同时选择多个适用标签，用英文逗号分隔（如：`complaint, angry`）
- 若无任何标签适用，输出"无"
- 只输出标签 key（英文），不输出中文含义或解释
